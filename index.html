How It Works

This system uses classical Operations Research methods to solve the LRT passenger assignment problem — minimizing travel cost (time) while satisfying supply (train capacity) and demand (passenger load) constraints.

🗺️

Transportation Problem

We model each LRT station as a source (supply) or destination (demand). The cost matrix represents travel time between stations in minutes.

↖️

Northwest Corner Method

NWCM provides the initial basic feasible solution by starting from the top-left cell and allocating as much as possible — fast but not optimal.

🎯

Hungarian Algorithm

We then apply the Hungarian Method (MODI / Stepping Stone) to reduce total cost iteratively until the optimal assignment is reached.

📡

Real-Time Simulation

The optimal assignments are simulated as live passenger flows between origin and destination stations with queue status indicators.

The 5-Step Process
1
Define Matrix — Choose source & destination stations, input supply, demand, and cost values.
2
NWCM Initial — Compute the initial basic feasible solution using the Northwest Corner Method.
3
Assign Tasks — Map each supply node to its best-fit demand node from the initial BFS.
4
Hungarian Optimization — Reduce the cost matrix and find the optimal one-to-one assignment.
5
Simulate Flow — Watch optimized passenger flows animate across the station network in real time.

Northwest Corner Method Step 2
Starting from the top-left (northwest) corner, allocate as much as possible to each cell before moving right or down. This gives us the initial basic feasible solution.
Task Assignments Step 3
Based on NWCM, each source station is assigned a destination. Non-zero allocations represent active passenger flows.
Hungarian Optimization Step 4
The Hungarian Algorithm finds the optimal one-to-one assignment by reducing the cost matrix row-by-row and column-by-column, then covering zeros with minimum lines.
